# Docker compose

### Цель задания

Научиться упаковывать приложения в контейнеры и организовывать их взаимодействие между собой.

### Условия

В рамках данного задания вам необходимо упаковать два Python-приложения в два Docker-контейнера и сделать так, чтобы первое приложение отвечало на запрос второго приложения по внутренней сети, а второе - отвечало на запросы внешних пользователей.

Для выполнения задания рекомендуем придерживаться следующих этапов:

- Создайте два Python-приложения на FastAPI. Первое должно по GET-запросу возвращать любой JSON-ответ. Второе по GET-запросу должно делать запрос к внешнему ресурсу и возвращать его ответ
- Создайте два Dockerfile, в которых будут описаны механизмы сборки приложений, и попробуйте каждое из них собрать в контейнер. Проверьте, что каждое приложение, упакованное в контейнер, работает
- Опишите в файле docker-compose.yaml сборку обоих приложений на основе их файлов Dockerfile, задайте приложениям имена и разные порты, а также пропишите общую сеть Возможный вариант (здесь приложения поменяны местами):

    ```
    services:
    app1:
    build: ./app1  # Путь к Dockerfile для 1-го приложения
    ports:
        - "8000:8000" # Для доступа извне, если нужно
    environment:
        APP2_HOST: app2
        APP2_PORT: 5000 # Порт, на котором работает app2
    depends_on:
        - app2  # Убедитесь, что app2 запущено перед app1


    app2:
    build: ./app2  # Путь к Dockerfile для 2-го приложения
    expose:
        - "5000"  # Порт в сети Docker, не для доступа извне
    environment:
        DEBUG: "True"
    ```

- Укажите во втором приложении в качестве источника адрес и порт первого приложения в общей сети
